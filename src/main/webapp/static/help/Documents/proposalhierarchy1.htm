Proposal Hierarchy

 

The Proposal Hierarchy section of the Proposal Actions page allows you to create hierarchies of multiple proposal development and budget documents,  link child proposals and budget types to current Proposal Development document hierarchies, and synchronize child/parent information.

Table 95 Proposal Development Document > Proposal Actions Page > Proposal Hierarchy Section –  Field/Button Descriptions

Field / Button

Description

Link to hierarchy button

Click to link an unlinked proposal document to a child proposal.

Create hierarchy button

Click to create a proposal hierarchy with the currently selected proposal and link child

Link Child Proposal

Click the lookup icon to search for and return the value to populate the box automatically, or if known, type the value in the box.

Link Budget Type

Use the drop-down menu by clicking the down arrow to display the list, and then click on an item in the list to highlight and select it to populate the box with your selection.

Link child proposal button

Click to link your selection in the Link Child Proposal field with the currently-accessed Proposal (which is the Parent Document).

Sync all button

Synchronize data displayed for all hierarchies that involve this newly established relationship.

Remove from hierarchy button

Remove the currently selected budget or proposal row from this hierarchy.

Sync to parent button

Synchronize child data to the parent proposal document to update data displayed therein.
