Time/Date

Read only

 

The date and time KC requested this action

Annotation

Read only

 

Notes from users who made changes to the document

 

Users with appropriate permissions can make changes to the document during the workflow.  Annotations enable them to make note of the changes.  The note is displayed in the Annotation field of the Route Log.

KC can generate an annotation.  An example is when someone blanket approves a document.  KC adds a message about the blanket approval in the Annotation field.

Node

Read only

 

The name of the workflow node

Examples of route nodes are INITIATED, PENDING APPROVE and COMPLETED.

Priority

Read only

 

The priority of the action

When a workflow rule is added, and a person is added to the rule, a priority is added to the person.  1 is highest.

Approval Policy

Read only

 

The approval policy for this action

Approval policies are:

Single Approval Required:

Forced Action

Read only

 

The indication of whether this action was forced

KC forces some actions.  An example is forcing remaining actions to a status of Complete when a user blanket approves a document. 

Your institution may set up workflow to force certain actions.

True: The action was forced.

False: The action was not forced.
