Editing A Proposal Development document While in Workflow

 

Q:  Can I still update my narrative document after I submit the proposal in to approval routing?

A:  Yes.  A user with a role containing the modify_narrative permission is able to use the “replace” functionality to upload a newer version of a proposal or internal attachment during routing.  The caveat here is that you can’t add new documents, but you can replace documents with newer versions as long as you uploaded a draft document prior to engaging workflow.

Q:  What happens if a faculty just needs to correct a misspelled word in the title?  Do I have to reject or disapprove the proposal?

A:  No.  Again, once a proposal is enroute, users with a role containing the “Alter Proposal Data” permission can update certain proposal data by accessing the Proposal Data Override panel on the Proposal Actions tab and ‘overriding’ the data that was originally entered.  What data can be overridden is configurable as well.  The table controlling what data is available to be ‘overridden’ is configured by using the “Proposal Dev Editable Columns” maintenance document.
