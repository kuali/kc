Custom Data

Your institution can elect to add custom data elements that are unique to your implementation of KC.  In order to collect institution-specific fields of information that are to be added to each Negotiation e-doc record, you must be able to modify the data in these institution-specific fields. 

 

Required Fields:  Some fields in the Custom Data tabbed section may be required for saving the Negotiation e-doc, depending on your institution’s unique implementation.

 

For general information about using the shared Custom Data functionality, see “Custom Data” on page 102 in Overview > Commone E-Doc Sections.

 

Consult your KC System Administrator for more information.  For technical setup details, Custom Attributes for the Negotiation document are established and configured by using the KC Custom Attribute functionality accessible from the Shared group on the Maintenance menu.  For more information, see “Custom Attribute Document” on page 1394.
