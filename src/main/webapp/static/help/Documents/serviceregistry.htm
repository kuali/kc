Service Registry

 

The Service Registry workflow screen displays a read-only view of the Kuali Service Bus (KSB) Service Registry, which includes Current Node information, and administrative function buttons to Refresh Service Registry and Delete localhost Entries, and a table that lists Published Services.  It is essentially used to view all services that are exposed on the Service Bus and includes information about them (such as IP Address, endpoint url, etc.). The Service Registry also includes an administrative function which allows the administrator to refresh its view of the registry.

Figure 2246 Service Registry Workflow Screen Example

[Refresh Service Registry]:  Click this button to update the list of published services.

[Delete localhost Entries]:  Click this button to delete local machine services from the table.

The following three subsections are displayed in table format:

•      Published Services:  Services in use by the local machine.

•      Published Temp Services:  Temporary services that are the result of Object Remoting.

For more information, see the Object Remoting section of the KSB Technical Reference Guide.

•      All Registry Services:  Published and temporary services that are available for the local machine.
