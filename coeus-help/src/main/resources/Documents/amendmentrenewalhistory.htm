Amendment/Renewal History

The Amendment/Renewal History section records all amendments, renewals and renewals with amendment that have been created for the protocol. It serves to give the user a real time snapshot of all past and current amendments and renewals created for the protocol in a single location i.e. within the original protocol. Administrators and Reviewers often need to review different amendment/renewals filed on a protocol, especially when multiple amendments and renewals are outstanding. The Amendment/Renewal History section displays high level information about the type of the amendment/renewal, the reason the amendment/renewal was created, the current status of the amendment/renewal and the create date and also provides access to additional details about the amendment/renewal submission.  The user can also directly click on the amendment or renewal hyperlink to access the amendment or the renewal protocol.

 

 The Amendment/Renewal History panel is only visible within the original protocol. For e.g. if the original protocol is 1405000028 and an amendment 1405000028A001 and a renewal without amendment 1405000028R001 have been created, the Summary & History section of only the 1405000028 protocol will contain the Amendment/Renewal History panel. The Summary & History section for the 1405000028A001 and 1405000028R001 protocols will only contain the Summary, Submission Details and History sub-sections.

 

 

Subsection

Description

Type

This column indicates if the entry in the Amendment/Renewal History sub-section is for an amendment or renewal without an amendment or a renewal with an amendment.

 

 Click on the hyperlinked value displayed within the Type column to access the entire amendment or renewal protocol.

Version Number

The version number column indicates the version number of the amendment/renewal protocol.

 

Note - Kuali Coeus increments versions when actions such as SMR/SRR/Return to PI/Withdraw are performed. For e.g. If an amendment protocol has been returned to the PI for revisions for the first time, the version number of the amendment protocol will be 1 and at that point of time, the Version Number column for the amendment protocol within Amendment/Renewal History will display 1.

Summary

This column displays the Summary description entered by the researcher at the time of amendment/renewal creation as to why he/she is creating the amendment or renewal.

Status

This column displays in real time the protocol status of the amendment/renewal protocol. As an amendment or renewal protocol moves through its lifecycle, the status column within the Amendment/Renewal History displays the real time status of the amendment/renewal protocol.

Created Date

This column displays when the amendment/renewal was created.

Amendment

/Renewal Summary panel

The Amendment/Renewal Summary panel within the original protocol’s Amendment/Renewal History sub-section displays the same Summary information that is provided within the Amendment or Renewal protocol’s Summary & History/Summary panel. 

 

The Summary panel displays high-level information about the amendment or renewal protocol such as protocol number, protocol dates, protocol title, PI name. In addition, it displays information about Personnel, Areas of Research, Attachments, Funding Source, Participant Types, Organization, Special Reviews and the Additional Information sub-panel within Protocol tab

Amendment

/Renewal Questionnaires panel

Any questionnaires deployed by institutions especially for amendments/renewals should be displayed within the panel with the answers provided by the researcher. In additional, the questionnaires pertaining to the original protocol are also displayed within this panel
