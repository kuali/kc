Amendment/Renewal History

The Amendment/Renewal History section records all amendments, renewals and continuations that have been created for the protocol. It serves to give the user a real time snapshot of all past and current amendments/renewals/continuation created for the protocol in a single location i.e. within the original protocol. Administrators and Reviewers often need to review different amendment/renewals/continuations filed on a protocol, especially when multiple amendments/renewals/continuations are outstanding. The Amendment/Renewal History section displays high level information about the type of the amendment/renewal/continuation, the reason the amendment/renewal/continuation was created, the current status of the amendment/renewal/continuation and the create date and also provides access to additional details about the amendment/renewal/continuation submission.  The user can also directly click on the amendment/renewal/continuation hyperlink to access the amendment or the renewal protocol.

 

 The Amendment/Renewal History panel is only visible within the original protocol. For e.g. if the original protocol is 1405000001 and a continuation protocol 1405000001C001 has been created, the Summary & History section of only the 1405000001 protocol will contain the Amendment/Renewal History panel. The Summary & History section for the 1405000001C001 protocol will only contain the Summary, Submission Details and History sub-sections.

 

 

Subsection

Description

Type

This column indicates if the entry in the Amendment/Renewal History sub-section is for an amendment/renewal or continuation protocol.

 Click on the hyperlinked value displayed within the Type column to access the entire amendment/renewal or continuation protocol.

Version Number

The version number column indicates the version number of the amendment/renewal or continuation protocol.

Note - Kuali Coeus increments versions when actions such as SMR/SRR/Return to PI/Withdraw are performed. For e.g. If an amendment protocol has been returned to the PI for revisions for the first time, the version number of the amendment protocol will be 1 and at that point of time, the Version Number column for the amendment protocol within Amendment/Renewal History will display 1.

Summary

This column displays the Summary description entered by the researcher at the time of amendment/renewal or continuation protocol creation.

Status

This column displays in real time the protocol status of the amendment/renewal or continuation protocol.

As an amendment/renewal/continuation protocol moves through its lifecycle, the status column within the Amendment/Renewal History displays the real time status of the amendment/renewal or continuation protocol.

Created Date

This column displays when the amendment/renewal or continuation protocol was created.

Amendment

/Renewal/Continuation Summary panel

The Amendment/Renewal/Continuation Summary panel within the original protocol’s Amendment/Renewal History sub-section displays the same Summary information that is provided within the Amendment/Renewal/Continuation protocol’s Summary & History/Summary panel. 

 

The Summary panel displays high-level information about the amendment or renewal protocol such as protocol number, protocol dates, protocol title, PI name. In addition, it displays information about Personnel, Areas of Research, Attachments, Funding Source, Organization, Special Reviews, Additional Information, Three R’s, Species/Groups, Procedures and Protocol Exceptions sub-panel within Protocol tab
