Custom Data

Custom Data is a tabbed section that appears in multiple areas of KC on e-doc pages.  It features functionality that is shared by multiple functional modules.  Multiple tabbed sections may make up an entire tabbed page of an e-doc that is labeled as Custom Data.  In a nutshell, it is a way for implementing institutions to collect and track fields of information that are unique to that institution – that is, different from and in addition to the “out-of-box” KC functionality.  Your institution has the ability to define custom attributes for the purpose of collecting additional information according to your unique institutional requirements.  This is done via the creation and maintenance of Custom Attributes via maintenance documents.

For more information about configuration of Custom Data for your institution, see “Custom Attribute” on page 1289 and “Custom Attribute Document” on page 1293 in Maintenance Documents > Shared.

Figure 71  Custom Data Page – Tabbed Section Examples

Groups are used to define which Custom Data fields are displayed on which e-docs.  One Custom Data tabbed section may display different fields on one e-doc than another. 

Required Fields:  Some fields may be required and are marked as such with the standard asterisk * in the user interface.  Depending on your implementation, some fields must be populated for saving the document, while some may be required for routing the document.
