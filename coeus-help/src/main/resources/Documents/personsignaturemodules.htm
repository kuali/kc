Person Signature Modules

The Person Signature Maintenance table allows institutes to create records associating the person’s name to the person’s scanned signature - so that it can be applied to correspondences generated when the person is the logged in user performing the action. The Person Signature Modules maintenance table associates the person’s signature record in the Person Signature Maintenance table to a specific Kuali Coeus Application. It also allows the school to define if the person’s signature for a specific Kuali Coeus application is the default signature in which case it would be used for all correspondences in the application irrespective of who the logged in user is.

For e.g. if an institute’s Compliance office wanted to use to use Elba Byler’s signature for correspondences for an IRB protocol and Inez Chew’s signature on correspondences for an IACUC protocol, they would need to a signature record within Person Signature maintenance table for both Elba Byler and Inez Chew. Then they would need to associate the Person Signature record for Elba Byler to the IRB module via the Person Signature Modules maintenance table and associate the Person Signature record for Inez Chew to the IACUC module via the Person Signature Modules maintenance table.

Note – As of 5.2.1, only IRB and IACUC applications have the Person Signature functionality.

Create New and Edit Person Signature Modules

The Create New action allowed on Person Signature Module allows you to select a person signature record from the Person Signature module record and associate to the Kuali Coeus module (e.g. IRB, IACUC etc. ). The Edit action allows you to maintain the person signature module record going forward.

 

 

 

Table 704 Person Signature Modules Maintenance Document

Field

Description

Person Signature Code

Use the lookup icon  to select the record from the Person Signature Maintenance table OR directly enter the code into the Person Signature Code field.

Module Code

Select the Module using the dropdown or the lookup icon to associate the person’s signature selected in the Person Signature Code field to a specific Kuali Coeus application.

Default Module Signature

Indicate whether the person’s signature should be the default for the module.

 

For e.g. Consider the scenario where users Della Woods and Elba Byler have signature records in the Person Signature table and these records are linked to the IRB module through the Person Signature Modules maintenance table and Elba Byler’s record in the Person Signature Modules maintenance table has the Default Module Signature checked.  In such a case, even if Della Woods is the person performing the IRB approval action, the approval correspondence will have Elba Byler’s signature because of the default indication.

 

Click within the checkbox to select the item.  A check mark appears within the checkbox to indicate the item is selected.  Click within the checkbox again to clear the mark and deselect the item.

Active

Indicate whether the Person Signature Modules record is active. Click within the checkbox to select the item.  A check mark appears within the checkbox to indicate the item is selected.  Click within the checkbox again to clear the mark and deselect the item.
