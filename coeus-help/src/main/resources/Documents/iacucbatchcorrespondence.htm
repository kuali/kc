IACUC Batch Correspondence

 

IACUC Batch Correspondence functionality provides a means for institutions to configure automated reminder notifications to go out to researchers to remind them on a timely basis of certain actions they need to take related to their protocols. The Kuali Coeus system provides Batch Correspondence functionality for the following two scenarios.

 

Ø Protocol Renewal Reminders – Batch correspondences related to renewal reminders serve to remind researchers when they need to renew their protocol because their protocol’s expiration date is approaching.

 

Ø Reminder to IACUC Notifications – Batch correspondences related to Reminder to IACUC notifications remind researchers to revise and re-submit their protocols when it has been returned for SMR (Specific Minor Revisions) or SRR (Substantive Revisions Required)

 

 In addition to generating reminders for the researcher, the Batch Correspondence also allows institutions to set up a final action to be taken on the protocol by the system at the time of the final reminder.

 

The following sections will describe how Batch Correspondence functionality works within Kuali Coeus. For help documentation related to setting up the Batch Correspondence, refer to the Maintenance help documents for IACUC Batch Correspondence. For help documentation related to running Batch Correspondence, refer to section on Batch Correspondence within the IACUC Committee help documents.
