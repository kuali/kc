3.

Citizenship

Found in the PI’s Person Extended Attributes

Citizenship Type

VALID Selection Options:

•      C. U.S. Citizen or noncitizen national

•      A, Non-US Citizen with temporary U.S. visa

•      N. Permanent Resident of U.S.

•      P. (Permanent Resident of the U.S. Pending)

Information Required for Local Implementation:

Every person will need to have a citizenship selection in their Person Extended Attributes since it is required for several GG forms.

If your organization is using a custom (external) data solution, the parameter PI_CITIZENSHIP_FROM_CUSTOM_DATA should be maintained to 0; alternatively, set this parameter to 1 to utilize the maintenance document values for Citizenship type which populate the Citizen Type field within the Person Extended Attributes.

 

Form End Notes:

Citizenship info for the form can come from an external warehouse table or from the KC Person Extended Attributes field of Citizenship Type [source determined by the parameter (PI_CITIZENSHIP_FROM_CUSTOM_DATA)].  If your organization is using a custom (external) data solution, the parameter PI_CITIZENSHIP_FROM_CUSTOM_DATA should be maintained to 0; alternatively, set this parameter to 1 to utilize the maintenance document values for Citizenship type which populate the Citizen Type field within the Person Extended Attributes.

Prior Form Versions:

The 1-2 version is being phased out in 2014 as the sponsor opportunities are retired or replaced. If using this or prior form versions, the narrative requirements are supported for the Inclusion Enrollment Report and the Targeted/Planned Enrollment, which have been converted to individual forms and no longer attachments to the 2-0 form.

The 1-0 version was retired by the sponsor with the required use of the new Research Strategy narrative. 

The 1-1 version of the form did not recognize Citizenship type of P. (Permanent Resident of the U.S. Pending)
