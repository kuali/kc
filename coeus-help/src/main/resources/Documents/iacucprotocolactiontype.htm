IACUC Protocol Action Type

 

 

The IACUC Protocol Action Type maintenance document is used to maintain existing IACUC protocol action codes and names.  Each name for an action has a numeric value associated with it, and the out-of-box product contains approximately 51 of them. 

These records are numeric codes and textual description values which are stored in database tables.  This data serves as reference information that is made available for display and selection in various areas of the system, including other e-docs.

The IACUC Protocol Action Type maintenance table provides multiple controls associated to protocol actions. Flags associated to each protocol action determines whether the IACUC protocol action will trigger a correspondence letter, will be used as a final action for batch correspondence and if it will trigger a submission action.

 

Existing Value Modification:  Existing values may be edited as long as the description is synonymous with the original term.  New values can be added.  You should not delete existing values.

Trigger Submission

Select if you want this action, when taken, to also cause a submission action to occur.

Trigger Correspondence

Select if you want this action, when taken, to also cause a correspondence letter to be generated by the IACUC protocol action.

Final Action for Batch Correspondence

Select if you want this action, when taken, to also cause a final disposition action for batch correspondence.

 You can check this Box to make an action appear within the “Take this Action” column for Protocol Renewal Reminders and Reminder to IACUC Notifications. However the Kuali Coeus system currently can execute only Close, Abandon, Expire and Suspend actions as a final action for Batch Correspondence. For any other action that needs to be final action for Batch Correspondence, code customization will be needed.
