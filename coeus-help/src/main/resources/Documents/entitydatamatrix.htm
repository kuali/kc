Entity Data Matrix

The Entity Data Matrix maintenance document (Financial Entity Data Matrix Maintenance Document) is used to establish and maintain numeric codes and textual description values which are stored in database tables.  This data serves as reference information that is made available for display and selection in various areas of the system, including e-docs and other maintenance e-docs.

Edit Entity Data Matrix Section

The Edit Entity Data Matrix section of the Entity Data Matrix maintenance document is a unique section that allows you to specify and maintain the details associated with the reference table record in fields.

 

Table 821 Entity Data Matrix Maintenance Document - Edit Entity Data Matrix Section Field Descriptions

Field

Description

Column Name

Required.  Type the name of the column in uppercase letters (for example, ‘CONSULTANT’).

Column Label

Required.  Type the title of the column as you wish it to appear in the user interface using title capitalization (first initial capitals for each major word – for example, ‘Consulting Service’).

GUI Type

Required.  Use the drop-down menu to select Dropdown or Checkbox for the desired Graphical User Interface (GUI) option selection field type.

Lookup Argument

Required.  Type lowercased text separated by underscore (for example, ‘age_range’)

Data Group Id

Required.  Type the suspected value in the box and click the direct inquiry icon or click the lookup icon to search for and return the value to populate the box automatically.

Status Flag

Required.  Click within the checkbox to select the item.  A check mark appears within the checkbox to indicate the item is selected.  Click within the checkbox again to clear the mark and deselect the item.

Column Sort Id

Required.  This is a numeric value that controls the sequential order in which the record option appears in a horizontal table layout.
