Proposals

 

The Proposals group on the Maintenance menu houses links to maintenance documents that are used to create and maintain database table records that are used by the Proposal functional module in the KC system.  The data values in these maintenance documents are made available for use in e-docs such as the Proposal Development document and the Budget document.  Within a standard e-doc, lookup and dropdown list fields exist in sections that allow for the searching for and selecting of data value options that are maintained using these maintenance documents to populate them when preparing and completing the e-doc.

Lookup/Create New Concept:  Typically, maintenance document links (bullet list of underlined text within labeled group boxes on the Maintenance menu) take you by default to Lookup screens that contain a create new  button at the top, right of the screen.  The lookup screen is used to optionally enter and/or select search criteria and then click a search  button to retrieve existing records in a search result table below the search criteria area of the screen.  The search result table contains an Action column at left that then allows you to take actions such as edit to access the existing maintenance document.  To create a new maintenance document, clicking the create new button allows you to initiate and display a new, blank maintenance document.

Figure 1715 Proposal Group on Maintenance Menu

 

 

Some of the links listed in this group are out of alphabetical order (for example, Proposal Log Status, Proposal Log Type, and To Be Named Person.  Some links in this group may also appear in other groups (for example, Object code also appears in the Shared group).  Some link names may take you to documents that are named something completely different (for example, Person Document Type takes you to the Science Code Lookup screen).
