Copy to New Document

 

The Copy to New Document section of the Protocol Actions page provides you with a copy protocol button.  Clicking it creates a new Protocol document with a unique Document Id.  In the header of the new document, an additional field displays the Id of the Protocol document being copied.  The data elements and content of the original are included in the newly created copy, and the new copy’s status is Pending/In Progress, allowing you to edit the new document as desired.

Figure 1214 Protocol Document, Document Header, Copied from Document Id Example

 

Roles & Copying:  In order to copy a Protocol document, you must have a role associated with the source Protocol document (for example, Aggregator, Viewer, IACUC Administrator, or IACUC Reviewer).  The person performing the copy action becomes the new Aggregator.

 

Status & Copying:  Protocol documents may be copies regardless of status, including a status that prevents modification of the protocol (for example, Approved or Submitted).

 

Items Not Copied:  Attachments, Questionnaire responses, Notes, Approval Dates, and Ad Hoc Routing are not copied from the source.
