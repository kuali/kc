Workflow Group

 

The Workflow group on the Researcher, Unit, and Central Admin menus displays links to workflow-related functionality.  The Researcher menu contains fewer links.  The purpose is to display convenient links for important workflow actions for each user type.  For all user types, this includes the ability to set workflow preferences to customize the display and functionality of your Action List.  For Unit and Central Admin users, it also includes the ability to generate routing reports, look up rules, and view routing and identity management document type hierarchies.

Table 42 Workflow Group Link Descriptions

Link

Brief Description

Preferences

Takes you to the Workflow Preferences screen, where you are able to customize the display of your Action List.  This includes the ability to adjust refresh rate, page size, email notifications and delegate filters; to specify which fields are displayed; and to specify the colors associated with documents in each route status.

Routing Report

Takes you to the Routing Report screen, which allows you to select a rule template, refine criteria (for example, effective date and document type) and view results.  From the search results, you are then able to open a document by Document ID for the purpose of performing operations on the document.

Rules

Takes you to the Rule Lookup screen, which allows you to refine criteria and search for and view workflow rules, and their associated templates and document types.

Rule Quicklinks

Takes you to the Routing & Identity Management Document Type Hierarchy screen, which displays tabbed sections for each document type.  Within each document type section, you are able to search for rules by document type; view Document Type and Document Configuration information that includes permissions and workflow/responsibilities.  With sufficient permissions, you are also able to add rules.  Access to current routing rules organized hierarchically by Document Type allows you to view, add new, or edit existing workflow rules.

Unit & Central Admin Considerations:  While the standard end users of KC (Researchers) are able to set their own Action List preferences, there are many more Workflow settings that are configured using the KC user interface that some Unit and Central Administration users must be familiar with.  On their menu screens, their Workflow menu groups include links for setting up workflow preferences, running reports and setting up rules and workgroups within the Kuali Enterprise Workflow (KEW). Whereas individual users can change their own workflow preferences, the institution’s system administrators or the super users within the functional area are responsible for making changes to the workflow rules.
