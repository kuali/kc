Narrative Status

 

The Narrative Status maintenance document is used to establish and maintain numeric codes and textual description values which are stored in database tables.  This data serves as reference information that is made available for display and selection in various areas of the system, including other e-docs.  Narrative Statuses note the state of a narrative attachment during proposal routing.  You can mark narrative file attachments as “Complete” or “Incomplete” on the Abstracts & Attachments tab of the KC Proposal Development document. Narratives must be marked “Complete” before the proposal can be submitted into routing.

Existing Value Modification:  Since these values are related to proposal validations and Grants.gov submissions, it is recommended that you do not delete or modify the two existing values:  Complete and Incomplete.  Otherwise, existing values can be edited as long as the description is synonmous with the original term. You can add new values. You should not delete existing values because it impacts S2S.

Edit Narrative Status Section

The Edit Narrative Status section of the Narrative Status maintenance document is a unique section that allows you to specify and maintain the details associated with the reference table record in fields.

Table 813 Narrative Status Maintenance Document - Edit Narrative Status Section Field Descriptions

Field

Description

Narrative Status Code

Required.  Enter the value that uniquely represents the single status in the system that can be selected for a particular narrative.

Description

Required.  The textual representation of this single state that a narrative can be in.  The status name that is commonly used to refer to this status in written or verbal language.
